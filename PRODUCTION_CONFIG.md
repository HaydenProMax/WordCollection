# Production Environment Configuration

This file lists the configuration files used by the production environment.

Target environment:

```text
Release ref: v1.0.0 by default
App name: word-collection
App user: word-collection
App home: /opt/word-collection
App port: 8000
Database: word_collection
Service: word-collection
```

Production should deploy a stable tag, not the moving `test` branch.

## 1. Required Private Config

Create this file on the Ubuntu server:

```bash
sudo nano /etc/word-collection.env
```

Recommended content:

```bash
DOMAIN='your-domain.example.com'
DB_PASSWORD='replace_with_a_strong_production_password'
OPENAI_API_KEY='replace_with_your_api_key'
ENABLE_HTTPS='1'
REPO_URL='https://github.com/HaydenProMax/WordCollection.git'
APP_REF='v1.0.0'
```

Optional values:

```bash
OPENAI_MODEL='gpt-5.5'
OPENAI_BASE_URL='https://www.fhl.mom/v1'
MODEL_PROVIDER='openai'
```

Protect it:

```bash
sudo chmod 600 /etc/word-collection.env
```

Do not commit this file. It contains secrets.

## 2. Production Deploy Script

Script in this repository:

```text
scripts/deploy-production-ubuntu-24.04.sh
```

Run it from the repository directory on the server:

```bash
sudo bash scripts/deploy-production-ubuntu-24.04.sh
```

This wrapper fixes these production values:

```text
APP_NAME=word-collection
APP_USER=word-collection
APP_HOME=/opt/word-collection
APP_PORT=8000
DB_NAME=word_collection
DB_USER=word_collection
```

It reads `APP_REF` from `/etc/word-collection.env`, defaulting to `v1.0.0`.

## 3. Generated App Env

The deploy script generates:

```text
/opt/word-collection/app/.env
```

Usually do not edit this file directly. Edit `/etc/word-collection.env`, then rerun the deploy script.

## 4. Generated systemd Service

The deploy script generates:

```text
/etc/systemd/system/word-collection.service
```

Useful commands:

```bash
sudo systemctl status word-collection
sudo systemctl restart word-collection
sudo journalctl -u word-collection -f
```

## 5. Generated Nginx Config

The deploy script generates:

```text
/etc/nginx/sites-available/word-collection
/etc/nginx/sites-enabled/word-collection
```

If the production domain changes, update:

```text
/etc/word-collection.env
```

Then rerun:

```bash
sudo bash scripts/deploy-production-ubuntu-24.04.sh
```

## 6. Health Checks

```bash
curl http://127.0.0.1:8000/api/lookups
sudo systemctl status word-collection
sudo nginx -t
```

## 7. Release Update

To deploy a new production release:

1. Verify the release in staging.
2. Merge to `main`.
3. Tag the release.
4. Push `main` and the tag.
5. Update `/etc/word-collection.env`:

```bash
APP_REF='vX.Y.Z'
```

6. Rerun:

```bash
sudo bash scripts/deploy-production-ubuntu-24.04.sh
```
