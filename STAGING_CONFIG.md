# Staging Environment Configuration

This file lists the configuration files used by the server-side staging environment.

Target environment:

```text
Branch: test
App name: word-collection-staging
App user: word-collection-staging
App home: /opt/word-collection-staging
App port: 8001
Database: word_collection_staging
Service: word-collection-staging
```

## 1. Required Private Config

Create this file on the Ubuntu server:

```bash
sudo nano /etc/word-collection-staging.env
```

Recommended content:

```bash
DOMAIN='test.your-domain.example.com'
DB_PASSWORD='replace_with_a_strong_test_password'
OPENAI_API_KEY='replace_with_your_api_key'
ENABLE_HTTPS='0'
REPO_URL='https://github.com/HaydenProMax/WordCollection.git'
```

Optional values:

```bash
OPENAI_MODEL='gpt-5.5'
OPENAI_BASE_URL='https://www.fhl.mom/v1'
MODEL_PROVIDER='openai'
```

Protect the file:

```bash
sudo chmod 600 /etc/word-collection-staging.env
```

Do not commit this file. It contains secrets.

## 2. Staging Deploy Script

Script in this repository:

```text
scripts/deploy-staging-ubuntu-24.04.sh
```

Run it from the repository directory on the server:

```bash
sudo bash scripts/deploy-staging-ubuntu-24.04.sh
```

This wrapper fixes these staging values:

```text
APP_NAME=word-collection-staging
APP_USER=word-collection-staging
APP_HOME=/opt/word-collection-staging
APP_PORT=8001
APP_REF=test
DB_NAME=word_collection_staging
DB_USER=word_collection_staging
```

## 3. Generated App Env

The deploy script generates:

```text
/opt/word-collection-staging/app/.env
```

It contains:

```text
APP_ENV=production
DATABASE_URL=postgresql+psycopg://word_collection_staging:<password>@127.0.0.1:5432/word_collection_staging
MODEL_PROVIDER=openai
OPENAI_MODEL=gpt-5.5
OPENAI_BASE_URL=https://www.fhl.mom/v1
OPENAI_API_KEY=<secret>
```

Usually do not edit this file directly. Edit `/etc/word-collection-staging.env`, then rerun the deploy script.

## 4. Generated systemd Service

The deploy script generates:

```text
/etc/systemd/system/word-collection-staging.service
```

Useful commands:

```bash
sudo systemctl status word-collection-staging
sudo systemctl restart word-collection-staging
sudo journalctl -u word-collection-staging -f
```

## 5. Generated Nginx Config

The deploy script generates:

```text
/etc/nginx/sites-available/word-collection-staging
/etc/nginx/sites-enabled/word-collection-staging
```

If the staging domain changes, update:

```text
/etc/word-collection-staging.env
```

Then rerun:

```bash
sudo bash scripts/deploy-staging-ubuntu-24.04.sh
```

## 6. pip Mirror Config

This server previously had trouble with the Tsinghua PyPI mirror. Use a working mirror such as Aliyun:

```bash
sudo tee /etc/pip.conf > /dev/null <<'EOF'
[global]
index-url = https://mirrors.aliyun.com/pypi/simple/
timeout = 120
retries = 10
trusted-host = mirrors.aliyun.com
EOF
```

Verify:

```bash
sudo -u word-collection-staging /opt/word-collection-staging/app/.venv/bin/pip index versions alembic
```

## 7. Health Checks

```bash
curl http://127.0.0.1:8001/api/lookups
sudo systemctl status word-collection-staging
sudo nginx -t
```
